# Research best practices for troubleshooting coding problems

## Spend 30 minutes talking with Chat GPT to understand different perspectives on using AI in coding. When is it helpful and when is it not?

-It is useful when code developers are required to understand the meaning of the code and the intention.
-It is not helpful when code developers ask chatbots to generate the entire code of an ETL pipeline of yfinance. This results in longer time spent debugging due to a lack of understanding of the code context , with time spent on finding code culprits.

## When do you prefer using AI vs. searching Google?

**AI** - To synthesize complex information and to write boilerplate code. It can explain the meaning of certain error message and library functions and can refactor code.

**Google** - Assume that I am working with a library but the modules might be deprecated. Google can help me find the documentation and find forum discussions behind a bug.

## How do you decide when to ask a colleague instead?\*\*

- When I have been stuck at debugging code culprits for more than 45 minutes without any form of progress
- A colleague is reliable when google and AI lacks knowledge of the business domain
- Human thinking can help corroborate and remark on my architectural ideas

## What challenges do developers face when troubleshooting alone?

- Limited coding knowledge.
- Mental fatigue due to the cognitive overload from coding.
- Relying on AI for debugging causes developers not to understand the culprits for system failure
