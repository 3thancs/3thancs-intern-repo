# Which extensions did you install? Why?
* **React Developer Tools**: 
  * Verifies with the API pipeline to observe the **facilitation and retrieval of data**
  * Verifies my understanding of how data is stored in the **client and database**
  * Allow developers to check components such as the components if it is a front-end or back-end issue. For example, an empty array could define as a back-end issue, while a visual flickering of the UI website page defines as a front-end API issue
  * How authentication JWT tokens are stored in DevTools


* Redux DevTools:
  * Traces precisely the **timeline** of when my API has failed. For example, an API displayed an error message of error 500 at 2.29 PM
  * Detects **impossible logic** in the API. For example, a sales number of -500 was fetched using SQL - which indicates a back-end issue


* JSON Viewer:
  * To debug **hidden API** responses. This occurs in scenarios where the endpoint returns data, but the front-end team report errors
  * Highlights syntax and type to verify data types
  * To view **nested** data more easily. For example, I am working on an e-commerce site using an API, where I can expand fields such as items.


* Lighthouse:
  * Validation tracks the **speed performance thresholds**. For example, I can validate the metric of the interaction of elements in the API. The time taken for after tracking is much longer than before tracking - which indicates a failed status 
  * Figure out how to process large real-world data in **development and production** based on the duration to process it.
